6º Semestre · 2026-1

O Projeto Destaque: Thunderstone

Dentre as seis APIs desenvolvidas ao longo do curso, o projeto do 6º semestre Thunderstone, em parceria com a Tecsys foi escolhido para detalhamento nesta apresentação.

6ª API · Contexto

Thunderstone — Visão Geral

Sistema de análise de mercado e criticidade para redes elétricas de distribuição.

🏢Empresa Parceira

Tecsys do Brasil empresa especializada em soluções de sensoriamento e eficiencia para redes elétricas de distribuição.

🎯Problema

Dimensionar e indicar as fiações elétricas que possuem o potencial de mercado para instalação de sensores Tecsys, baseado nas falhas em redes de média e alta tensão no Brasil, a partir de dados regulatórios da ANEEL (DEC, FEC).

💡Solução

Plataforma ingere dados abertos da ANEEL (Geodatabase e CSVs), processa as camadas geoespaciais e regulatórias via pipeline ETL assíncrono, calcula índices de criticidade por conjunto elétrico e formula relatório PDF com dashboard e mapa de calor georreferenciado para apoiar a decisão comercial.

🗄️Persistência

MongoDB para dados regulatórios processados (BDGD, DEC, FEC) e PostgreSQL para usuários, credenciais e autenticação.

⚙️Processamento

Tarefas distribuídas via Celery com Redis como broker, permitindo download, descompactação e processamento em paralelo dos arquivos GDB.

6ª API · Minha Contribuição

Pipeline ETL de Ingestão (ANEEL)

Foco principal: construção do pipeline ETL assíncrono de ingestão dos dados da ANEEL.

1
/etl/download-gdb → task_download_gdb: download via streaming (httpx), validação do ZIP e encadeamento automático da próxima etapa via assinatura Celery, com backoff exponencial em falhas de rede.
2
task_descompact_gdb: descompacta o ZIP, localiza o .gdb, valida o schema das camadas obrigatórias (CTMT, CONJ, SSDMT) e dispara um chord Celery com as tasks de processamento em paralelo.
3
task_processar_ctmt e task_processar_conj: limpeza, validação e estruturação dos registros de conjuntos elétricos (energia mensal e perdas por classe tarifária), persistidos no MongoDB.
4
/etl/load-dec-fec → task_load_dec_fec_realizado/limite: download de CSVs, processamento em chunks de 10.000 registros e bulk upsert idempotente no MongoDB.

6ª API · Desafios Técnicos

Desafios Técnicos

🧩Decomposição do Pipeline

Arquivos GDB podem ter gigabytes e camadas com decenas de milhares de geometrias, isso exigiu pensar a solução em etapas independentes e combináveis (download, validação, extração e processamento paralelo por camada) em vez de um processo monolítico.

⚡Processamento Paralelo

Uso do padrão chord do Celery para processar CTMT, CONJ e SSDMT em paralelo e agregar os resultados de forma confiável. A camada SSDMT, com maior volume de geometrias, suporta paralelismo configurável por chunks.

🛡️Resiliência e Observabilidade

Cada task foi construída com logging estruturado, backoff preparado para falhas de rede, validação antecipada de schema e idempotência via bulk upsert, evitando arquivos mal-formado da ANEEL que quebram a pipeline.

Análise Crítica

Pontos Negativos da Metodologia API

⏳Início Precoce

Nos dois primeiros períodos, a metodologia não faz pleno sentido: o aluno recém-ingressado ainda não possui base teórica para fazer perguntas verdadeira elucidativas, combinado com modelo pedagogico passivo, sobrecarga de alunos por alunos.

🤐Disciplinas externas a API desconetadas

Disciplinas importantes que poderiam ser apoio à API e ao empreendorismo estão desconexas da atualidade e da dinâmica da API. Ex: Legislação aplicada à tecnologia da informação

📉Subutilização

Alunos que já sabem programar sentem-se subutilizados ou sobrecarregados: A busca pela faculdade também é para consolidar conceitos teóricos e estruturais, não apenas para codificar.

Análise Crítica

Proposta de Melhoria

🔁Proposta de Transição

O 1º e 2º período deveriam ser consolidadores de conceitos com práticas correlacionadas. O 3º período ensaio da API com a CADI SJC sob orientação. Para então 4º a 6º semestre, clientes externos reais.

📝Justificativa

Atualmente, a assimetria de conhecimento entre os alunos nos primeiros semestres faz com que muitos recorram a IAs generativas apenas para entregar código funcional, sem absorver o aprendizado esperado pela metodologia.
